# Glossary

> **閱讀指引**：標有 🔑 的術語是初學者**必讀**的核心概念，建議優先理解。標有 📚 的術語可以在需要時再查閱，**初學者可先略過**。

| Term | 中文說明 | 白話解釋 | 在本書中的位置 | 優先度 |
|---|---|---|---|---|
| Prompt Engineering | 以 prompt 技巧提升模型輸出的做法 | 就像你學會怎麼「問對問題」讓 AI 給出更好的答案。重點在於措辭和技巧。 | 第 1 章 | 🔑 必讀 |
| Harness Engineering | 將模型包裝進可操作、可限制、可測試系統的工程工作 | 不只是跟 AI 聊天，而是幫 AI 蓋一個「工作環境」，讓它能安全、可控地完成任務。就像幫一匹馬套上韁繩（harness），你才能控制它的方向和力道。 | 第 1-3 章 | 🔑 必讀 |
| Runtime | 執行 turn loop 的核心 | 系統的「總指揮」，負責控制每一輪對話的流程：接收輸入 → 呼叫模型 → 處理工具 → 回傳結果。就像交響樂的指揮，控制所有人什麼時候該做什麼。 | 第 6 章 | 🔑 必讀 |
| Tool Registry | 集中管理工具宣告與可用性的層 | 一本「工具清冊」，記錄系統有哪些工具可用、每個工具怎麼用、需要什麼權限。就像公司的設備管理系統，要借什麼設備都得先從這裡查。 | 第 7 章 | 🔑 必讀 |
| Permission Policy | 控制工具權限的規則層 | 決定「誰能做什麼」的規則。就像大樓門禁系統——有些人只能進大廳，有些人能進辦公室，要根據權限等級判斷。 | 第 8 章 | 🔑 必讀 |
| Session | 對話與狀態的持續容器 | 系統的「記憶」。如果沒有 session，每次對話都像第一次見面，之前說過什麼全忘光。Session 把對話記錄保存下來，讓系統能「接續上次的進度」。 | 第 9 章 | 🔑 必讀 |
| Prompt Assembly | 將 system prompt 與 context 組裝起來的流程 | 在真正送給 AI 之前，把各種背景資訊、指令、上下文「組裝」成一份完整的提示詞。就像廚師備料——不是隨便丟進鍋裡，而是按順序、按比例組合好。 | 第 10 章 | 📚 進階 |
| MCP | 外部能力擴充協定與工具橋接方式 | 讓系統能接上「外部插頭」的標準介面。就像 USB 接口——任何符合規格的設備都能接上使用，不需要為每個設備重新設計接口。 | 第 12 章 | 📚 進階 |
| Parity Harness | 用 mock / scenario 驗證行為一致性的測試方法 | 一種測試方式：用預先寫好的劇本（scenario）去跑系統，檢查結果是否和預期一致。就像餐廳用「神秘客」測試服務品質是否達標。 | 第 14 章 | 📚 進階 |
| Mini Harness | 本書後半段用 Python 重建的最小系統 | 你自己動手做出來的縮小版 harness。它不是玩具，而是保留了核心骨架的教學版系統，讓你真正理解 harness 怎麼運作。 | 第 16-19 章 | 🔑 必讀 |
