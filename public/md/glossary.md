# Glossary

| Term | 中文說明 | 在本書中的位置 |
|---|---|---|
| Prompt Engineering | 以 prompt 技巧提升模型輸出的做法 | 第 1 章 |
| Harness Engineering | 將模型包裝進可操作、可限制、可測試系統的工程工作 | 第 1-3 章 |
| Runtime | 執行 turn loop 的核心 | 第 6 章 |
| Tool Registry | 集中管理工具宣告與可用性的層 | 第 7 章 |
| Permission Policy | 控制工具權限的規則層 | 第 8 章 |
| Session | 對話與狀態的持續容器 | 第 9 章 |
| Prompt Assembly | 將 system prompt 與 context 組裝起來的流程 | 第 10 章 |
| MCP | 外部能力擴充協定與工具橋接方式 | 第 12 章 |
| Parity Harness | 用 mock / scenario 驗證行為一致性的測試方法 | 第 14 章 |
| Mini Harness | 本書後半段用 Python 重建的最小系統 | 第 16-19 章 |
